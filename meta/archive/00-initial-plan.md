# План книги «Веб-разработка для тех, кто не знал, куда смотреть»

*Рабочее название — поменяем.*

---

## Краткие вводные

**Аудитория:** абсолютные новички — как в книге 3 о формах 1С. Человек открывает книгу с пустой головой и нулевыми ожиданиями к стеку.

**Сквозной пример:** «Канцтовары онлайн». Та же фиктивная компания, что и в книге 3, только теперь у неё появляется публичный сайт + онлайн-приложение для заказа. Это даёт двойную выгоду: читатель-1С-ник получает преемственность, читатель-не-1С-ник — конкретный осязаемый объект, который мы строим по ходу книги.

> Если для веб-аудитории Канцтовары не зайдут — скажи, заменим (вариант: студия дизайна, барбершоп, или абстрактная «контора»).

**Структура модуля:** название → 4–7 параграфов (§X.Y) → итог модуля. В каждом параграфе — теория + кусочек развития примера.

**Прикидка по объёму:** 12 модулей, ~65 параграфов, ~140–160 тыс. знаков (как книга про ООП в 1С).

## Что я переставил относительно майндмэпа и почему

1. **«Анатомия Браузеров» переехала** из ветки `WEB-серверы` в модуль 4 (Браузер как клиент). В исходнике это явная путаница — браузер живёт на клиенте.
2. **«Авторизация» и «Сертификаты» собраны в HTTP-модуле**, а не размазаны по другим веткам — родной дом этих тем для новичка.
3. **«Серверная генерация» слита со «Способами рендеринга»** в модуль 7. В майндмэпе они отдельно с пунктирной связью; для новичка логичнее одна цельная глава «как из запроса получается HTML».
4. **«Типы сайтов» подняты в модуль 1** — это вводный контекст, а не отдельная глава ближе к концу.
5. **«Проектирование» и «Дистрибюция» слиты в финальный модуль** — по отдельности они на 2–3 параграфа, на отдельный модуль не тянут.
6. **«История развития» сжата** до §1.1: одного параграфа достаточно, отдельный модуль для новичка — перебор.

## Мелочи в майндмэпе, которые стоит поправить

- **API: `wsl`** — судя по контексту, это либо **WebSocket (WS)**, либо **WSDL (SOAP)**. Сейчас читается как Windows Subsystem for Linux 🙂
- **OODB: `Cache`** → InterSystems **Caché** (с é). Это бренд.
- **OODB: `db40`** → почти наверняка **db4o**.
- **«Серверa приложений»** — опечатка, должно быть «серверы».
- **«Сесия»** → «сессия».
- **«Объектно орентированные»** → «ориентированные».
- **«Flexbox?»** со знаком вопроса — в плане я вынес во вкладку CSS-инструментов (§10.4 вместе с Grid).
- **HTTP-методы**: «GET, POST, Другие» — стоит явно перечислить PUT/PATCH/DELETE, для REST они важны.
