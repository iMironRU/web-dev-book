# «Между строк» — код сквозного примера

Книжный интернет-магазин, который растёт вместе с книгой. Здесь живёт
его код.

## Стадии и теги

Код развивается стадиями. Каждая стадия — git-тег. Чтобы увидеть
состояние ровно на нужном витке:

```
git checkout stage/03-vitrina
```

| Тег | Стадия | Появляется около |
|---|---|---|
| `stage/01-zaglushka`    | «сайт в разработке», один HTML | Модуль 2 |
| `stage/02-lending`      | одностраничный лендинг          | Модуль 2–3 |
| `stage/03-vitrina`      | каталог из шаблонов + БД        | Модуль 6–7 |
| `stage/04-vitrina-blog` | + блог, SEO, три читателя       | вплетается |
| `stage/05-magazin`      | корзина, заказы, авторизация    | Модуль 8 |
| `stage/06-zhivoy`       | живой интерфейс (SPA)           | Модуль 9 |
| `stage/07-integracii`   | оплата, доставка, 1С            | Модуль 10 |
| `stage/08-prod`         | домен, хостинг, CI/CD           | Модуль 11 |

## Принцип

Стадия рождается, когда в книге наступает её **боль**, а не когда «по
жизни так строят сайт». Порядок стадий подчинён оси болей книги, не
логике стройки. Подробнее — `../CLAUDE.md`, раздел про «Между строк».

Структуру кода (один прогрессирующий проект против папок-снапшотов)
дорабатываем — пока договорились на теги.
