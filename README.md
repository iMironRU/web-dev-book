# Веб глазами древнего человека

> Рабочее название, поменяем.

Учебник по веб-разработке для тех, кто открывает тему с нуля. Не
справочник «делай так», а книга про то, **почему** веб устроен именно
так — через боль, из которой родилась каждая технология, и через
бытовые вещи, которые человек делал тысячи лет.

Автор: imiron.ru (iMironRU). Статус: **в работе, скелет**.

## Идея в двух абзацах

Каждая технология вводится не как факт, а как ответ на конкретную
невыносимость предыдущего шага. Машинка не умеет жирный → разметка.
Оформление руками в пятистах местах → CSS. Файл лежит у меня, а Вася в
Перми его не видит → серверы, HTTP, DNS. И так до конца.

Под каждой технологией — древний предок: HTTP-запрос это бланк
требования в библиотеке, база данных — картотека ЗАГСа, CORS — шрам от
реальных краж денег. Сквозной вопрос книги: «ты это сейчас пишешь для
кого — для поисковика, для браузера или для человека?»

## Сквозной пример

Через всю книгу строится **«Между строк»** — книжный интернет-магазин.
Он растёт стадиями (заглушка → лендинг → витрина → магазин → …), каждая
стадия помечена git-тегом `stage/NN-...`. Можно сделать `checkout`
нужной стадии и увидеть код ровно на том витке, до которого дочитал.
Код — в `example/`.

## Структура репозитория

```
.
├── CLAUDE.md          рабочая память проекта (читать первым)
├── book.yml           манифест: порядок глав, метаданные, форматы
├── manuscript/        сама книга, файл на параграф
│   ├── 00-vvedenie/
│   ├── 01-sayt-eto-chto/
│   └── ...             NN-MM-slug.md + README.md модуля
├── meta/              «как мы пишем» — не для читателя книги
│   ├── spine.md            финальная спина (цепочка болей)
│   ├── style-guide.md      стилевая записка
│   ├── roots-and-pains.md  метод: корни + боли роста
│   └── archive/            как мы пришли (исходный план, майндмэп)
├── example/           код магазина «Между строк» (по стадиям/тегам)
├── assets/            иллюстрации, схемы, скриншоты
└── scripts/           scaffold.py — генератор дерева и book.yml
```

## Конвенции

- Имена файлов — транслит, без точек: `02-03-html-skobki.md`.
- Один параграф — один файл. Один параграф ≈ одна посиделка читателя
  (8–12 тыс. знаков, 1–1,5 часа с упражнениями).
- Статус параграфа — во front-matter: `todo → draft → review → done`.
- Порядок глав живёт **только** в `book.yml`, который генерирует
  `scripts/scaffold.py`. Руками список файлов не править.

## Как пересобрать скелет

```
python3 scripts/scaffold.py
```

Существующие `.md` не перезаписываются (написанный текст в безопасности).
`book.yml` пересобирается всегда. Чтобы добавить/переставить параграф —
правьте манифест `MODULES` в `scripts/scaffold.py`, затем запустите его.

## Как собрать книгу для чтения

Нужен [pandoc](https://pandoc.org/). Затем:

```
make epub      # EPUB
make pdf       # PDF (нужен XeLaTeX)
make html      # одностраничный HTML
make all       # всё сразу
```

Артефакты складываются в `dist/` (в git не попадает). Ночные сборки
собираются автоматически — см. `.github/workflows/nightly.yml`.
